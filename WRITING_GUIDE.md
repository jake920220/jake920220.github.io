# Writing Guide (준현의 기록)

이 문서는 블로그 글 작성 시 리노가 따라야 하는 규칙이다.

## 공통
- 문체는 기본적으로 **한다/이다** 체로 통일한다. (서평/기술 동일)
- 불필요한 감탄/과장은 줄이고, 결정/근거/트레이드오프를 중심으로 쓴다.
- 내부 링크/파일 경로/스크립트명은 가능한 한 정확히 표기한다.
- 섹션 제목은 가능하면 **어미를 떼고 명사형으로 끝낸다.**
  - 예: "오탐 줄이기: 키워드만으로는 부족하다" → "오탐 줄이기: 키워드만으로는 부족"
  - 예: "운영 관점: 세션 컨텍스트와 크론은 분리한다" → "운영 관점: 세션 컨텍스트와 크론은 분리"
- 기술 포스팅은 섹션 컨텍스트가 바뀌는 지점에서 **H2(##) 전후로 공백을 넉넉히** 두어 문단이 뭉쳐 보이지 않게 한다.

## 기술(tech) 포스팅 규칙
- 카테고리: `tech`
- 썸네일/이미지 파일 위치 규칙:
  - `public/images/tech/<slug>/...`
  - 예: `public/images/tech/openclaw-review-automation-part1/openclaw-cover.jpg`
- 포스트 frontmatter의 `image`는 위 경로를 사용한다.
- 카드 썸네일에서는 좌상단에 **기술** 배지를 표시한다.
- 기술 포스팅 썸네일 카드는 book-review처럼 책 프레임을 쓰지 않고, 이미지가 카드 영역을 **object-cover**로 채우는 형태를 기본으로 한다.

## 서평(book-review) 포스팅 규칙(요약)
- 섹션 전환마다 `##` H2 중제목
- 볼드 강조는 1~2개만
- 태그: `프로그래밍`, `IT`, `서평` + 출판사 태그 추가
- 카드 썸네일은 16:9 템플릿(책표지 contain + 종이 텍스처 + '서평' 배지)
