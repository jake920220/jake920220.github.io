# 커밋 규칙 (준현의 기록)

이 문서는 이 저장소에서 커밋 메시지를 작성할 때 지켜야 하는 규칙이다.

## 원칙
- **시멘틱한 단위로 커밋한다.** (UI/문서/리팩터/기능 등 목적이 섞이면 분리)
- 커밋 제목은 **한글로** 작성한다.
- 제목은 **명시적이고 짧게** 쓴다.
  - 어미/사족 금지: "~함", "~했음", "~합니다" 같은 표현을 피한다.
- 커밋 본문에는 변경 내용을 **목록으로** 적는다.
  - 왜/무엇/영향 범위를 짧게 남긴다.

## 승인 규칙(중요)
- 기본값: **커밋하지 않는다.**
- 변경은 작업 트리(unstaged) 상태로 유지하고, 준현님이 "커밋해줘"라고 **명시적으로 승인**했을 때만 커밋한다.
- 커밋 전에 가능하면 로컬에서 동작/스타일을 **실제 화면으로 검증**한다(예: Playwright/Lighthouse/수동 확인).

## 형식
- 제목(1줄)
- 빈 줄
- 본문(불릿 목록)

예시

```
블로그 소개 문구 정리

- 블로그 이름을 "준현의 기록"으로 통일
- 카테고리 표기를 Tech/Book Review로 정리
- 불필요한 섹션(기술스택/로컬실행) 제거
```
